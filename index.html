A modern graphical interface to pulsar timing packages

The most widely used pulsar timing packages, tempo and tempo2, are written in FORTRAN and C++. Programs written in these languages tend to be time-intensive to extend in terms of features. In the case of tempo2, a GUI is available in some plugins through the old PgPlot library.

Qtip aims to bring the functionality of tempo and tempo2 into the modern age, by using Qt and IPython. The package currently relies on libstempo as its timing engine, so the solutions are identical to when you are using tempo2 directly. Support for PINT is being added as PINT progresses in development.

Qtip is designed for scientist. Scientists don't want a ready-made program that solves a problem. They don't want to just push a button. They want to fiddle with stuff. They want to do things the programmer has not thought of. Qtip tries to accommodate these needs by providing a full IPython interface alongside a graphical interface. Solve for a pulsar using the Plk emulator, while retaining direct access to the libstempo object at all times. This approach allows one to be much more productive when solving pulsars!

DISCLAIMER

Qtip is still in the very early stages of development. The plk emulator has basic functionality, so it is possible to solve pulsars, but it is nowhere near feature-complete. If you are interested in helping out you are encouraged to fork the project on github!

Requirements

All of the requirements, except for tempo2, are available through pip
  • Python 2.7 (no Python 3 yet...)
  • numpy
  • tempo2
  • libstempo or PINT
  • scipy
  • matplotlib
  • PyQt
  • IPython 2.0
  • pygments
  • pyzmq
  • jdcal
